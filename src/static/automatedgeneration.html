Summary

The Hi-Audio platform integrates automated processing modules to support music analysis, metadata enrichment, and annotation workflows. These components assist researchers and musicians in building and exploring an open database of music corpora.

Uploaded audio files and associated metadata are automatically processed by backend services after submission. This processing pipeline may extract technical metadata, identify potentially copyrighted musical works using audio fingerprinting, generate machine-based annotation labels related to instrumentation and acoustic content, and compute additional signal descriptors for musicological and technical analysis.

These automated operations aim to improve dataset consistency, facilitate retrieval and analysis, and support reproducible research practices. The sections below describe the main automated processes currently deployed on the platform, together with the external services, libraries, and models involved in their implementation.

Metadata Extraction

The platform automatically extracts technical metadata from uploaded audio files. This may include, among others, information such as file format, codec, duration, sample rate, channel configuration, bitrate, and bits per sample.

This extraction is implemented using the pydub Python library. The resulting metadata is used to normalize audio asset descriptions and improve indexing, filtering, and corpus management.

Service used: Pydub

Copyright Identification via Audio Fingerprinting

The platform uses the AcoustID service to perform audio fingerprint-based identification of uploaded music tracks.

When a fingerprint match is obtained, the system may retrieve and associate metadata such as the identified artist name, track title, and confidence or matching score. This process assists in the detection and annotation of potentially copyrighted material. Content flagged through this process is subject to manual review and may be removed from the platform in cases of confirmed infringement.

Service used: AcoustID

Automated Annotation

The platform applies automated annotation methods to infer descriptors from uploaded audio material. These machine-generated annotations are intended to support corpus exploration, retrieval, and downstream musicological analysis.

Instrument Class Inference

The platform uses the AST model (Audio Spectrogram Transformer) to infer instrument-related classes from audio recordings. AST generates automated annotation outputs corresponding to musical instrument categories, together with associated confidence scores.

Library/model used: AST

Acoustic and Music Content Descriptors

The platform uses the Python bindings of Essentia to compute additional descriptors from uploaded audio files.

Depending on the processing configuration, these descriptors may include annotations or estimates related to:

  • presence of human voice,
  • tonality,
  • percussion-related content,
  • tempo (BPM),
  • root mean square energy (RMS).

These descriptors are automatically computed to enrich audio records with signal-level and music-relevant information. When the analysis detects the presence of human speech or non-singing voice content, the corresponding recordings may be flagged for manual review and potential removal in order to ensure compliance with applicable data protection regulations, including the GDPR.

Library used: Essentia

Random Default Profile Pictures

For user accounts that do not provide a custom profile picture, the platform may rely on Picsum Photos to generate random placeholder images.

This mechanism is used exclusively to assign non-sensitive default visual placeholders to user profiles. It is not part of the scientific processing pipeline applied to uploaded audio content, but is documented here for transparency regarding automated platform behavior.

Service used: Picsum Photos

Implementation Notes

The automated processing pipeline is executed by a backend background module that processes uploaded audio files after submission. Its purpose is to enrich records with technical metadata and machine-generated annotations in a systematic and reproducible manner.

In order to improve storage efficiency and optimize content delivery to end users, large uploaded audio files in formats such as WAV or FLAC may be automatically compressed on the server side. This process reduces file size and improves page loading performance when accessing audio content through the platform. During this step, audio files may be transcoded to the AAC codec using the pydub Python library. The resulting audio is stored in an MP4 container format (formally MPEG-4 Part 14) using the .m4a file extension. This conversion does not alter the original uploaded file, which remains preserved in the platform storage for reference and reproducibility purposes.

Further technical details on implementation and software components are available in the Hi-Audio backend repository: https://github.com/idsinge/hiaudio_backend
